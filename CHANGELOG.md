# Changelog

All notable changes to the Simple Button Generator plugin will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2024-01-XX

### Added
- Initial release of Simple Button Generator
- Core button generation functionality with 8 action types:
  - URL links
  - JavaScript functions
  - Email links
  - Phone calls
  - File downloads
  - Form submissions
  - Modal/popup triggers
  - Scroll to element
- Comprehensive styling options:
  - Custom color pickers for background, text, and border
  - Font family selection (9 font options)
  - Button sizes: Small, Medium, Large, Custom
  - Responsive button sizes: Small, Medium, Large, Extra Large
  - Box shadow effects (5 options)
  - Border customization:
    - Border styles: None, Solid, Dashed, Dotted, Double
    - Border widths: 1px to 5px
    - Border colors: Auto, predefined colors, custom color picker
    - Corner styles: Square, Rounded, More Rounded, Pill
- Advanced features:
  - Live preview with floating draggable panel
  - Real-time preview updates
  - Analytics tracking integration:
    - Google Analytics (gtag)
    - Google Analytics (ga - legacy)
    - Facebook Pixel
    - Custom event tracking
  - Custom CSS support with examples
  - CSS Examples Modal with copy-paste examples
  - Download HTML file functionality
  - Copy code to clipboard
- Comprehensive documentation:
  - README.md with full feature list
  - Installation guide
  - FAQ section
  - Action types guide
  - Analytics tracking guide
  - Border options guide
  - CSS examples guide
  - Testing guide
  - Publishing guide
- Test suite:
  - Master test runner
  - Backend tests
  - Frontend tests
  - Plugin functionality tests
  - Analytics tracking tests
  - Responsive size tests
  - Validation tests

### Security
- Proper nonce verification for AJAX requests
- Input sanitization and output escaping
- Protection against direct file access
- WordPress coding standards compliance

### Technical
- WordPress 4.0+ compatibility
- PHP 5.6+ support
- Modern browser compatibility
- Responsive design support
- Clean, semantic HTML output
- Well-structured CSS generation
